# AI 编程提示词：宇宙命理“星尘”积分系统与差异化权益技术规格 (v4)

**目标**：实现一个基于“全功能开放+积分制”的订阅体系，所有积分**永久有效**。

## 1. 数据模型定义 (Database Schema)

### 1.1 订阅计划表 (`subscription_plans`)
*移除 credit_validity_days 字段*
| 字段名 | 类型 | 描述 | 示例值 |
| :--- | :--- | :--- | :--- |
| `name` | VARCHAR(50) | 计划名称 | 'Free', 'Pro', 'Elite' |
| `ai_model_tier` | VARCHAR(50) | AI 模型等级 | 'gpt-4o-mini', 'gpt-4o', 'gpt-4o-priority' |
| `monthly_credit_quota` | INT | 每月赠送积分 | 0, 1200, 4000 |
| `earning_multiplier` | DECIMAL(3, 2) | 积分获取倍率 | 1.00, 1.50, 2.00 |
| `shop_discount` | DECIMAL(3, 2) | 商店折扣率 | 1.00, 0.90, 0.80 |
| `badge_type` | VARCHAR(50) | 身份标识 | null, 'silver_medal', 'gold_frame' |

### 1.2 积分账户表 (`stardust_accounts`)
*移除过期相关字段*
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `user_id` | UUID | 用户 ID |
| `balance` | INT | 当前总余额 |
| `total_earned` | INT | 累计获得积分 |
| `total_spent` | INT | 累计消耗积分 |

### 1.3 积分流水表 (`stardust_logs`)
*移除过期相关字段*
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `id` | BIGSERIAL | 流水记录 ID |
| `user_id` | UUID | 用户 ID |
| `action_type` | VARCHAR(50) | 行为类型 |
| `amount` | INT | 变动数额 |
| `created_at` | TIMESTAMP | 记录创建时间 |

## 2. 核心业务逻辑

### 2.1 动态 AI 模型路由逻辑
*逻辑不变*

### 2.2 积分获取算法 (`POST /api/stardust/earn`)
**逻辑更新**：
1. `final_amount = base_amount * user.subscription_plan.earning_multiplier`。
2. 记录流水时，`description` 需注明“包含 [X] 倍率加成”。
3. **取消** 积分有效期计算逻辑。

### 2.3 积分扣除逻辑
*逻辑不变，但无需考虑过期问题*

## 3. 业务规则汇总

| 维度 | 星尘游客 (Free) | 星系探索者 (Pro) | 宇宙主宰 (Elite) |
| :--- | :--- | :--- | :--- |
| **AI 模型** | gpt-4o-mini | **gpt-4o** | **gpt-4o (高优先级)** |
| **每月配额** | 0 | **1,200** | **4,000** |
| **获取倍率** | 1.0x | **1.5x** | **2.0x** |
| **积分有效期** | **永久有效** | **永久有效** | **永久有效** |
| **商店折扣** | 无 | **9 折** | **8 折** |

## 4. 额外技术要求

1. **原子性**：所有积分变动操作必须是原子性的。
2. **防作弊**：在处理 `referral_success` 行为时，必须在后端校验被邀请人的状态。
3. **用户通知**：在积分消耗成功或失败时，应触发一个小弹窗告知用户。
4. **取消过期任务**：**移除**任何关于积分过期检查和扣除的定时任务。
