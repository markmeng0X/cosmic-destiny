1.

推荐建立ttclid后端映射 ​

​

📌

ttclid在EAPI环境中需要从前端获取并储存（具体方式参考后下方[User相关参数说明](https://bytedance.sg.larkoffice.com/docx/XZM4dFuLkoNgy2xbonMlN3higrh#share-NOtIdY9NNokX7IxauuGlYq9Ig1e)）​

非常建议建立对应后端映射：与用户相关参数：uid进行映射，以保证在后续多次转化事件中，服务器回传事件都可以携带对应ttclid帮助归因；否则，很可能存在ttclid丢失情况；​

大致映射逻辑：​

1.

前后端用户uid参数相同的情况下，在前端获取对应ttclid并储存在后端；​

2.

在后端建立对应映射机制：MySQL或者Redis缓存；​

3.

事件回传时取对应ttclid参数：建议双保险：在前端获取不到ttclid时从数据库中寻求对应映射数据；​

4.

建立ttclid刷新机制：ttclid会根据计划设置的归因窗口存在不同的时效性，需要定期更新；​

​

一、回传参数层级：event\_id （优先推荐）​

​

|     |     |
| --- | --- |
| 在回传事件的时候携带 event\_id 参数，event\_id的逻辑可以自己定义，目的为使得事件触发以某特定逻辑（时间，设备，触发频次）进行对事件的去重。​<br>请务必验证event\_id的逻辑，不要写死，否则对事件归因和最后计划的表现有很大影响。​<br>如果同一事件配置了web 和 server 回传，确保同一事件在web 和 server的 event\_id 相同，避免同一事件重复。​<br>常见event\_id的逻辑为get cookie；​<br>可以参考此文档：[Docs \| TikTok API for Business](https://business-api.tiktok.com/portal/docs?id=1739584864945154)​<br>​ | Web Pixel示例​<br>​<br>![Feishu Docs - Image](blob:https://bytedance.sg.larkoffice.com/26bf5008-bbcf-44cd-99d4-9141b4e0e53d)<br>​<br>EventsAPI 示例：​<br>​<br>![Feishu Docs - Image](blob:https://bytedance.sg.larkoffice.com/95c65817-d7e1-41de-8cd3-34b45945758b)<br>​ |

​

二、网页逻辑层级 （按需设置）​

如果event\_id层级无法满足对事件去重维度的需求，可以在网页逻辑层级自定义去重逻辑。并不推荐：因为event\_id层级的去重在TikTok数据后台可追踪/溯源，可以从TikTok方面检查配置，出现类似问题也更容易发现。网页层级的逻辑无法直接检查，存在问题更难排查。​

示例：​

​

Code block​

JavaScriptCopy

// 事件去重函数​

functionhandleEvent(eventId) {​

// 检查 localStorage 中是否已有该事件的记录​

const eventKey = \`event\_${eventId}\`;​

​

// 如果事件已记录，跳过处理​

if (localStorage.getItem(eventKey)) {​

console.log(\`事件 ${eventId} 已处理过，跳过此次处理。\`);​

return;​

}​

​

// 如果没有记录，处理事件并记录该事件​

console.log(\`处理事件 ${eventId}...\`);​

​

// 将事件ID存储到 localStorage 中，设置一个过期时间（例如：1小时后）​

const expireTime = Date.now() \+ 3600000; // 当前时间 \+ 1小时（3600000毫秒）​

localStorage.setItem(eventKey, expireTime);​

​

// 你可以在这里执行事件处理的代码，例如发送请求等​

// sendEventToServer(eventId);​

}​

​

// 清除过期事件​

function cleanUpExpiredEvents() {​

const keys = Object.keys(localStorage);​

​

keys.forEach(key => {​

// 找到所有的事件ID记录​

if (key.startsWith("event\_")) {​

const expireTime = localStorage.getItem(key);​

​

// 如果事件过期，清除它​

if (expireTime && Date.now() > expireTime) {​

console.log(\`事件 ${key} 已过期，清除它。\`);​

localStorage.removeItem(key);​

}​

}​

});​

}​

​

// 示例：触发事件​

handleEvent("user\_click\_button\_123");​

handleEvent("user\_click\_button\_123"); // 第二次触发时会跳过​

​

// 定期清理过期的事件记录​

setInterval(cleanUpExpiredEvents, 60000); // 每分钟检查一次过期事件​
